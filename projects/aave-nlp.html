AAVE Analysis Using NLP

African American Vernacular English · Natural Language Processing · Pratt Institute MS DAV

Overview

This project explores how well machine learning models capture African American Vernacular English (AAVE), using comedy movie scripts as a test case. The core problem is that most NLP systems are trained on standardized English, which means dialects like AAVE are often poorly understood and represented.

Non-Black individuals emulating the speech of Black people is pervasive. This serves as an inquiry into the content of popular comedic media. My role was to design and execute a complete workflow that spanned from data collection through analysis to interpretation. Key stakeholders include researchers, technologists, and anyone building or evaluating NLP systems, especially those interested in bias, representation, and language equity. Success meant quantitatively and qualitatively showing gaps in how AAVE is represented in model-relevant text spaces.

What makes this problem unique is that AAVE is highly contextual and culturally embedded. It is inconsistently documented in structured datasets, and often misclassified or erased in NLP pipelines. External constraints included limited AAVE corpora, reliance on scraped scripts and social media data, and a semester timeline.

Approach

I approached this as a comparative NLP pipeline examining how AAVE appears across two domains: movie scripts (structured, written dialogue) and social media text (tweets). As I started my analysis, I knew that embeddings and cosine similarity are commonly used to measure semantic alignment, but I also suspected they would fail to capture syntactic nuance. I conducted research on how AAVE is treated in NLP literature, built an AAVE lexicon to detect surface-level usage, and considered domain adaptation as a possible method of improvement.

This led to two key analytical angles: lexical analysis (are AAVE terms present at all?) and semantic similarity (do models treat AAVE-aligned text as meaningfully similar?).

Workflow

The dataset was built from two main sources: a large set of text files containing comedy movie scripts and a corpus of tweets. Since the tweet data originally came in JSON format, it was converted into a tabular structure for easier processing. Both datasets were then standardized into a comparable format before analysis.

I used a dictionary-centered approach to identify AAVE presence, iterating through the text with n-grams to capture both single-word and multi-word expressions. This produced lexical metrics like density and variation as a baseline measure of how often AAVE appeared in each dataset. 


          # cleaning the tweet list to remove special characters (e.g., #, @, emojis)

              tweet = ' '.join(tweet.split())            # remove extra whitespace

          d = defaultdict(list)

              k = i.split()
              d[len(k)].append(tuple(k))

          def ngram_counter(my_text_split, d):
              match_counts = dict()
              for n,v in d.items():
                  c = Counter(ngrams(my_text_split, n))
                  for k in v:
                      if k in c:
                          match_counts[k] = c[k]
              return match_counts

After establishing that lexical baseline, I shifted to semantic analysis. Embeddings were generated for both datasets using a pretrained model that converted each script as a single embedding, tweets individually then averaged for a stable comparison point.

This produced two layers of insight: a single similarity score (script vs. corpus average) and a distribution of similarities (script vs. each individual tweet). The second layer revealed variability in whether a script aligns strongly with some tweets but not others.

A number of technical challenges required iteration: handling mismatched data structures, reshaping embeddings for cosine similarity compatibility, and scaling the pipeline across multiple scripts. Small details like avg_tweet_embedding = np.mean(tweet_embeddings, axis=0).reshape(1, -1) ended up being critical for valid results. The lexical density analysis also proved more limited than expected. Many AAVE expressions are highly contextual in ways that standard token-based matching doesn't capture well.

Results

The pipeline produced consistent, reproducible metrics across all ten scripts. AAVE presence is uneven and often superficial in films outside of culturally specific contexts. Even when AAVE terms appear, semantic similarity scores remain relatively low which suggests models are not capturing deeper linguistic structure.

Key Points

AAVE is underrepresented in model training data, topic consistently overrides dialect in embedding space, and dialectal distinctions collapse into a single semantic space regardless of cultural specificity. Next steps could include audio transcription (e.g., via Whisper), expanding to other minoritized Englishes, or experimenting with domain adaptation.

What I Learned

  • Lexical presence ≠ semantic understanding
  • Embedding models can obscure cultural and linguistic nuance
  • Data representation matters just as much as model choice
